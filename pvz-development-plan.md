# 🌻 网页版植物大战僵尸 — 开发计划

## 技术栈 (Tech Stack)

| 层级 | 选择 | 理由 |
|------|------|------|
| 渲染 (Rendering) | HTML5 Canvas | 适合 2D 游戏，性能好 |
| 语言 (Language) | 原生 JavaScript (Vanilla JS) | 无框架依赖，适合学习 |
| 打包 (Bundling) | 单 HTML 文件 | 简单部署，方便迭代 |
| 素材 (Assets) | Canvas 绘图 + Emoji | 无需外部图片资源 |

---

## 开发阶段总览

```
阶段1: 游戏骨架 → 阶段2: 核心机制 → 阶段3: 战斗系统 → 阶段4: 完整游戏
  (画布+网格)      (种植+阳光)      (僵尸+碰撞)      (关卡+UI+音效)
```

## 阶段 1：游戏骨架 (Game Skeleton)

**目标**: 画出游戏场景，实现基本的网格系统和游戏循环 (Game Loop)

### Prompt 1.1 — 游戏画布与网格

```
创建一个 HTML5 Canvas 植物大战僵尸游戏的基础框架：

技术要求：
- 单个 HTML 文件，内含 CSS 和 JS
- Canvas 尺寸 900×600，居中显示
- 绘制 5 行 9 列的草地网格（每格 80×100px）
- 左侧留出种植栏区域（120px宽），顶部留阳光计数区域
- 网格交替深浅绿色，模拟草坪效果
- 用 requestAnimationFrame 实现 60fps 游戏循环

代码要求：
- 面向对象设计：Game 类管理全局状态，Grid 类管理网格
- 每行每列要有清晰的坐标映射函数（像素坐标 ↔ 网格坐标）
- 加详细中文注释
```

### Prompt 1.2 — 游戏状态管理

```
在上一步的基础上，添加游戏状态管理系统：

- GameState 枚举：MENU / PLAYING / PAUSED / WIN / LOSE
- 状态切换逻辑和简单的开始菜单界面
- 实现暂停/继续功能（按 P 键或点击暂停按钮）
- 帧率显示（调试用，可开关）
- 全局计时器：记录游戏已进行时间（秒）

保持代码结构清晰，用 class 组织。
```

## 阶段 2：核心机制 (Core Mechanics)

**目标**: 实现阳光系统、植物种植、植物卡牌栏

### Prompt 2.1 — 阳光系统 (Sun System / 阳光系统)

```
实现阳光收集系统：

阳光机制：
- 初始阳光值 50，显示在左上角
- 天空每 5-8 秒随机掉落一个阳光（值 25）
- 阳光从顶部随机位置缓慢下落，到达随机高度后停留 8 秒消失
- 点击阳光后，阳光飞向计数器（带动画：贝塞尔曲线飞行）
- 收集音效用 Web Audio API 生成简单的"叮"声

视觉效果：
- 阳光用 Canvas 绘制：黄色圆形 + 光芒射线，带旋转动画
- 阳光悬停时有微微放大效果
- 收集时有粒子散射特效

用 Sun 类管理每个阳光实例，SunManager 类管理生成和回收。
```

### Prompt 2.2 — 植物卡牌栏 (Seed Packet Bar / 种子卡牌栏)

```
实现左侧植物选择卡牌栏：

卡牌系统：
- 卡牌栏在画面左侧，纵向排列
- 每张卡牌显示：植物图标（用 Canvas 绘制的简笔画或 emoji）、
  名称、阳光消耗值、冷却进度条
- 初始可用植物：向日葵(50)、豌豆射手(100)、坚果墙(50)、樱桃炸弹(150)
- 卡牌冷却机制：种下后进入冷却（灰色遮罩 + 倒计时扫描动画）
- 阳光不足时卡牌变暗，不可点击

交互流程：
1. 点击卡牌 → 卡牌高亮，鼠标变成植物预览（半透明跟随鼠标）
2. 鼠标移到有效网格 → 格子高亮提示
3. 点击放置 → 扣除阳光，播放种植动画，启动冷却
4. 右键或按 Esc → 取消种植

用 SeedPacket 类和 PlantFactory 工厂模式管理。
```

### Prompt 2.3 — 植物基类与向日葵 (Plant Base & Sunflower)

```
实现植物系统的基类和第一个植物——向日葵：

Plant 基类：
- 属性：gridX, gridY, health, maxHealth, type
- 方法：update(), draw(), takeDamage(), die()
- 死亡动画：植物缩小并变灰消失
- 血量条显示在植物上方（受伤时才显示）

向日葵 (Sunflower)：
- 消耗 50 阳光，冷却 7.5 秒
- 每 20 秒产生一个阳光（在自身位置弹出）
- 外观：用 Canvas 绘制——棕色圆形花盘 + 黄色花瓣（8片椭圆），绿色茎
- 空闲动画：花瓣轻微摇摆（正弦波驱动）
- 产出阳光时有"弹跳"动画

PlantManager 类管理所有已种植的植物实例。
```

## 阶段 3：战斗系统 (Combat System)

**目标**: 实现射击、僵尸、碰撞检测

### Prompt 3.1 — 豌豆射手与子弹 (Peashooter & Projectile)

```
实现豌豆射手和子弹系统：

豌豆射手 (Peashooter)：
- 消耗 100 阳光，冷却 7.5 秒
- 外观：绿色圆形头部 + 管状嘴巴 + 茎和叶子（Canvas绘制）
- 当同一行有僵尸时才开始射击（检测逻辑）
- 每 1.5 秒发射一颗豌豆
- 射击动画：头部轻微后仰

豌豆子弹 (Projectile)：
- 绿色小圆球，匀速向右飞行（速度 200px/s）
- 飞出画面右边界后销毁
- 命中僵尸后销毁，造成 20 点伤害
- 命中特效：小型绿色溅射粒子

ProjectileManager 管理所有子弹，注意性能——用对象池(Object Pool)模式复用子弹对象。
```

### Prompt 3.2 — 僵尸系统 (Zombie System)

```
实现僵尸系统：

Zombie 基类：
- 属性：x, y, row, health, speed, damage, state
- 状态机：WALKING → EATING → DYING → DEAD
- 从右侧屏幕外生成，沿所在行向左移动

普通僵尸 (BasicZombie)：
- 血量 200，速度 20px/s，攻击力 10/s
- 外观（Canvas绘制）：灰绿色头部、破烂衣服、僵硬走路动画（手臂前伸，腿部交替）
- 走路动画：身体轻微上下晃动 + 手臂摇摆
- 接触植物时切换为 EATING 状态（咬合动画）
- 死亡动画：倒地 + 身体变灰 + 消失

路障僵尸 (ConeheadZombie)：
- 血量 380（路障吸收 180），速度同上
- 头顶有橙色三角锥，锥被打掉后变为普通僵尸

碰撞检测：
- 僵尸前进时检测前方是否有植物（矩形碰撞）
- 与子弹的碰撞检测（圆-矩形碰撞）

ZombieManager 管理所有僵尸实例。
```

### Prompt 3.3 — 波次系统 (Wave System / 波次系统)

```
实现僵尸波次生成系统：

WaveManager：
- 关卡配置用 JSON 对象定义，例如：
  {
    waves: [
      { time: 10, zombies: [{ type: "basic", row: 2, delay: 0 }] },
      { time: 30, zombies: [
        { type: "basic", row: 1, delay: 0 },
        { type: "basic", row: 3, delay: 2 },
        { type: "conehead", row: 2, delay: 4 }
      ]},
      ...
    ],
    finalWave: 5  // 最后一波的索引
  }
- 每波开始时屏幕右侧显示 "一大波僵尸正在接近..." 警告文字
- 最终波更密集，有特殊红色警告
- 波次进度条显示在屏幕顶部

胜负判定：
- 所有波次僵尸被消灭 → WIN
- 任何僵尸到达最左侧 → LOSE（僵尸吃掉你的脑子动画）
```

## 阶段 4：完整游戏 (Full Game)

### Prompt 4.1 — 更多植物 (More Plants)

```
添加更多植物种类：

坚果墙 (Wall-nut)：
- 消耗 50 阳光，冷却 30 秒
- 血量 4000（超高），纯防御，不攻击
- 外观：棕色椭圆坚果，上面有裂纹表情
- 三段受损外观：完好(>66%) → 裂纹(33%-66%) → 严重破损(<33%)
- 僵尸啃食时有颤抖效果

樱桃炸弹 (Cherry Bomb)：
- 消耗 150 阳光，冷却 50 秒
- 种下后 1 秒爆炸，消灭 3×3 范围内所有僵尸
- 爆炸动画：红色膨胀 → 橙色冲击波 → 屏幕轻微震动(screen shake)
- 爆炸后留下焦黑痕迹（3秒后消失）

寒冰射手 (Snow Pea)：
- 消耗 175 阳光，冷却 7.5 秒
- 发射蓝色冰豌豆，命中后僵尸减速 50%，持续 3 秒
- 减速效果视觉：僵尸变蓝 + 脚下冰霜粒子
```

### Prompt 4.2 — UI 与特效打磨 (UI & Polish)

```
打磨游戏UI和视觉特效：

开始菜单：
- 游戏标题 "植物大战僵尸" 带卡通字体效果（Canvas绘制描边文字）
- 向日葵和僵尸的装饰动画
- "开始游戏" 按钮（悬停放大 + 发光效果）

游戏内 HUD：
- 顶部信息栏：阳光数量（带图标）、波次进度、暂停按钮
- 铲子工具：点击后可移除已种植的植物（返还 0 阳光）

胜利/失败画面：
- 胜利：烟花粒子特效 + "你赢了!" + 重玩按钮
- 失败：屏幕变红变暗 + "僵尸吃了你的脑子!" + 重试按钮

全局特效：
- 屏幕震动函数 screenShake(intensity, duration)
- 粒子系统 ParticleSystem 类（通用，支持不同颜色/大小/寿命）
- 伤害数字飘字效果（僵尸受伤时头顶弹出 "-20"）
```

### Prompt 4.3 — 音效系统 (Sound System)

```
用 Web Audio API 实现纯代码生成的音效（无需音频文件）：

SoundManager 单例类：
- 使用 AudioContext 和 OscillatorNode 合成音效
- 每个音效用频率、波形、包络参数定义

需要的音效：
- 收集阳光：短促上升音调，正弦波 (440Hz → 880Hz, 0.1s)
- 种植植物：低沉的 "咚"，三角波 (200Hz, 0.15s)
- 豌豆射击：快速 "噗"，白噪声脉冲 (0.05s)
- 僵尸受伤：钝击声，方波 (100Hz, 0.1s)
- 僵尸死亡：下降滑音 (300Hz → 50Hz, 0.3s)
- 爆炸：白噪声 + 低频震动 (0.5s)
- 胜利音乐：简单上升音阶 C-E-G-C
- 失败音乐：下降半音阶

提供音量控制和静音开关。
```

## 关键术语表 (Glossary)

| 英文 | 中文 | 说明 |
|------|------|------|
| Game Loop | 游戏循环 | 每帧执行 update → draw 的循环 |
| Sprite | 精灵 | 游戏中的可绘制对象 |
| Collision Detection | 碰撞检测 | 判断两个对象是否重叠 |
| Object Pool | 对象池 | 复用对象以减少垃圾回收 |
| State Machine | 状态机 | 用状态和转换管理对象行为 |
| Factory Pattern | 工厂模式 | 用统一接口创建不同类型的对象 |
| Bézier Curve | 贝塞尔曲线 | 用于平滑的运动路径 |
| Particle System | 粒子系统 | 管理大量小型视觉特效元素 |
| HUD | 平视显示器 | 游戏内的界面信息层 |
| Screen Shake | 屏幕震动 | 爆炸等事件的反馈特效 |
| Wave | 波次 | 一批同时或连续出现的僵尸 |
| Cooldown | 冷却时间 | 技能/卡牌使用后的等待时间 |

## 使用建议

1. **逐步开发**：严格按阶段顺序，每完成一个 Prompt 确保能运行再进入下一个
2. **单文件开发**：所有代码保持在一个 HTML 文件中，方便调试
3. **每步测试**：每个 Prompt 完成后手动测试所有功能
4. **迭代改进**：如果某个 Prompt 的结果不满意，追加描述修改需求，不要跳过
5. **保存版本**：每完成一个阶段，复制一份作为备份（如 `pvz-v1.html`, `pvz-v2.html`）

> 💡 **提示**: 你可以把每个 Prompt 直接发给 Claude，要求生成完整代码。建议一次只发一个 Prompt，拿到可运行的代码后再发下一个，并附上"在以下代码基础上继续"。
